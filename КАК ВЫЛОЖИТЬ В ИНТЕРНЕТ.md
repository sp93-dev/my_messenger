# Как выложить мессенджер в интернет (Render.com)

Всё уже подготовлено. Нужно только: залить проект на GitHub и подключить его к Render.
Django будет раздавать и сайт, и API, и WebSocket — один сервис. База — бесплатная PostgreSQL.

Итог: получишь публичный адрес вида `https://messenger-xxxx.onrender.com`, который
открывается с любого телефона и компьютера.

---

## Шаг 1. Залить проект на GitHub

1. Зайди на https://github.com → **New repository** (зелёная кнопка).
2. Имя любое (например `messenger`), сделай **Private** или **Public**, нажми **Create repository**.
3. Загрузи файлы проекта в этот репозиторий. Самый простой способ без командной строки:
   - На странице репозитория нажми **uploading an existing file**.
   - Перетащи туда **всю папку** проекта (`index.html`, `app.js`, `styles.css`, `api.js`,
     `render.yaml`, `.gitignore`, и папку `backend`).
   - Внизу нажми **Commit changes**.

   > Если умеешь работать с git — просто `git init`, `git add .`, `git commit`, `git push`.

⚠️ Файлы `backend/db.sqlite3`, `backend/media/`, `backend/staticfiles/` загружать НЕ нужно —
они уже в `.gitignore` и создадутся на сервере сами.

---

## Шаг 2. Создать сервис на Render

1. Зайди на https://render.com и зарегистрируйся (можно через GitHub-аккаунт — удобнее).
2. Нажми **New +** → **Blueprint**.
3. Выбери свой репозиторий `messenger` из списка → **Connect**.
4. Render увидит файл `render.yaml` и сам предложит создать:
   - **web-сервис** `messenger` (это Django),
   - **базу данных** `messenger-db` (PostgreSQL).
5. Нажми **Apply** (или **Create**).
6. Подожди 3–7 минут — Render установит зависимости, соберёт статику, применит миграции
   и запустит сервер. Статус станет **Live**.

Готово! Вверху страницы сервиса будет ссылка вида `https://messenger-xxxx.onrender.com` —
это и есть твой мессенджер в интернете. Открывай и регистрируйся.

---

## Что уже настроено за тебя

- ✅ `render.yaml` — описывает сервис и базу, Render всё поднимет сам.
- ✅ `SECRET_KEY` — Render сгенерирует надёжный автоматически.
- ✅ `DEBUG=false` — безопасный продакшен-режим.
- ✅ `DATABASE_URL` — подключение к PostgreSQL подставится автоматически.
- ✅ Django раздаёт сам сайт (`/`), скрипты, API (`/api/...`) и WebSocket (`/ws`).
- ✅ `api.js` сам определяет адрес — менять ничего не нужно.
- ✅ HTTPS Render даёт автоматически (нужно для звонков и WebSocket).

---

## Важно знать (особенности бесплатного тарифа)

- **Засыпание**: на free-тарифе сервис «засыпает» после 15 минут простоя. Первый заход
  после сна грузится ~30–60 сек, потом работает быстро. Лечится платным тарифом ($7/мес).
- **Загруженные файлы** (фото/видео в чате) на free-тарифе **стираются при перезапуске**
  сервиса (нет постоянного диска). Сообщения и аккаунты в PostgreSQL — сохраняются.
  Чтобы файлы не пропадали: подключи постоянный диск Render (платно) или облако (S3) — скажи, помогу.
- **Звонки** работают через интернет (PeerJS + STUN), оба собеседника должны быть онлайн.

## Свой домен (по желанию)

Если есть свой домен (например `mychat.ru`):
1. В Render открой сервис → **Settings** → **Custom Domains** → добавь домен.
2. У регистратора домена пропиши указанные Render DNS-записи.
3. HTTPS-сертификат Render выпустит сам.

---

## Если что-то пошло не так

- Открой сервис на Render → вкладка **Logs** — там видно ошибки.
- Частые причины: не залилась папка `backend`, или забыл закоммитить `render.yaml`.
- Можно прислать мне текст из Logs — помогу разобраться.

> Технические детали для ручного деплоя (daphne, nginx, Redis, S3) — в `backend/ДЕПЛОЙ.md`.
