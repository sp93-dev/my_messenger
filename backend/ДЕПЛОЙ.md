# Развёртывание бэкенда в продакшене

Бэкенд — Django + Channels (ASGI). Для реалтайма (WebSocket) нужен ASGI-сервер
(**daphne** или **uvicorn**), обычный gunicorn-WSGI WebSocket не отдаёт.

## 1. Подготовка
```bash
cd backend
python -m pip install -r requirements.txt
# для прода доустановите из закомментированных в requirements.txt:
python -m pip install whitenoise channels-redis "psycopg[binary]" uvicorn
```

## 2. Переменные окружения
Скопируйте `.env.example` → `.env`, заполните и экспортируйте:
```bash
export SECRET_KEY="длинная-случайная-строка"
export DEBUG=false
export ALLOWED_HOSTS="ваш-домен.com"
export DATABASE_URL="postgres://user:pass@localhost:5432/messenger"   # опционально
export REDIS_URL="redis://localhost:6379/0"                            # опционально (для нескольких процессов)
```

## 3. Миграции и статика
```bash
python manage.py migrate
python manage.py collectstatic --noinput
```

## 4. Запуск ASGI-сервера
**daphne:**
```bash
daphne -b 0.0.0.0 -p 8000 server.asgi:application
```
**или uvicorn:**
```bash
uvicorn server.asgi:application --host 0.0.0.0 --port 8000 --workers 4
```

> При нескольких воркерах обязательно задайте `REDIS_URL` — иначе WebSocket-события
> не дойдут между процессами (in-memory channel layer работает только в одном процессе).

## 5. Перед фронтендом
- В `api.js` поменяйте `API_BASE` на адрес вашего сервера (например, `https://api.ваш-домен.com/api`).
- Поднимите HTTPS (например, через nginx) — тогда WebSocket будет `wss://` автоматически
  (код сам подставляет `ws/wss` по протоколу `http/https`).
- Раздавайте папку фронтенда (`index.html`, `app.js`, `styles.css`, `api.js`) любым веб-сервером.

## 6. nginx (пример проксирования)
```nginx
location /api/  { proxy_pass http://127.0.0.1:8000; }
location /media/ { proxy_pass http://127.0.0.1:8000; }
location /ws    { proxy_pass http://127.0.0.1:8000;
                  proxy_http_version 1.1;
                  proxy_set_header Upgrade $http_upgrade;
                  proxy_set_header Connection "upgrade"; }
```

## Модерация (жалобы)

Сообщения с запрещёнными словами (терроризм, наркоторговля, оружие, CSAM —
список в `api/moderation.py`) **не блокируют** пользователя, а создают жалобу.
Админ получает уведомление и решает сам.

Кто видит жалобы — пользователи с флагом `is_staff`. Чтобы назначить админа:
```bash
python manage.py createsuperuser          # создать админа
# или повысить существующего:
python manage.py shell -c "from django.contrib.auth.models import User; u=User.objects.get(username='ИМЯ'); u.is_staff=True; u.save()"
```
Затем войдите этим аккаунтом в приложение — появится кнопка **🛡 Модерация**
с счётчиком новых жалоб и панелью (offender, категория, совпавшее слово, текст,
кнопка «Решено»). Все жалобы также видны в Django-админке (`/admin`).

Словарь запрещённого правится в `backend/api/moderation.py` — добавляйте/убирайте
термины под свою аудиторию (авто-действий нет, поэтому риск минимален).

## Чеклист безопасности прода
- [ ] `DEBUG=false`, секретный `SECRET_KEY` из окружения
- [ ] Конкретные `ALLOWED_HOSTS`
- [ ] Сузить CORS (`CORS_ALLOW_ALL_ORIGINS=False` + список доменов) — правится в `settings.py`
- [ ] PostgreSQL вместо SQLite
- [ ] Redis для channel layer при масштабировании
- [ ] HTTPS/WSS
