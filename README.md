# Мой любимый мессенджер 💬

Полноценный веб-мессенджер с **настоящим бэкендом** (Django + REST API + SQLite):
переписка синхронизируется между разными устройствами через сервер.

## Как запустить

### 1. Запустить бэкенд (сервер)
- Двойной клик по **`backend/Запустить сервер.bat`**
  (или вручную: `cd backend` → `python manage.py migrate` → `python manage.py runserver 127.0.0.1:8000`)
- Откроется окно с сервером на `http://127.0.0.1:8000`. **Не закрывайте его** во время работы.

### 2. Открыть приложение
- Двойной клик по **`index.html`** (или через локальный веб-сервер).
- Зарегистрируйтесь (имя + юзернейм + пароль).
- Чтобы пообщаться: на другом устройстве/в другом браузере откройте `index.html`,
  зарегистрируйте второго пользователя — и пишите друг другу по юзернейму.
  Сообщения идут через сервер и синхронизируются (опрос каждые ~2.5 сек).

> Требуется установленный Python с пакетами: `django`, `djangorestframework`, `django-cors-headers`
> (уже установлены в этой системе).

## Архитектура

- **Бэкенд** (`backend/`): Django + DRF + **Channels (WebSocket)**, токен-авторизация, SQLite.
  - На сервере: **аккаунты, пользователи, чаты, сообщения, истории, загруженные файлы**
    (общая база для всех устройств).
  - **Реалтайм через WebSocket** (`/ws`) — мгновенная доставка (~180 мс); опрос `/sync`
    остаётся подстраховкой и для онлайн-статусов.
  - **Файлы** грузятся на сервер (`/api/upload`, до 25 МБ) и отдаются из `/media/`.
  - Эндпоинты: `/api/register`, `/login`, `/logout`, `/me`, `/users`, `/presence`,
    `/chats`, `/chats/<id>`, `/chats/<id>/messages`, `/messages/<id>`, `/upload`,
    `/stories`, `/stories/<id>`, `/sync?since=`.
- **Фронтенд** (`index.html`, `app.js`, `styles.css`, `api.js`):
  - `api.js` — связь с сервером (fetch + токен + WebSocket + polling-fallback).
  - Локально (в `localStorage`) хранятся только **настройки устройства**: тема, папки,
    стикеры, черновики, PIN-код приложения и т.п.

## Продакшен

Бэкенд готов к деплою: настройки берутся из переменных окружения
(`SECRET_KEY`, `DEBUG`, `ALLOWED_HOSTS`, `DATABASE_URL` для PostgreSQL, `REDIS_URL`
для общего channel layer). Подробная инструкция — в **`backend/ДЕПЛОЙ.md`**,
зависимости — в `backend/requirements.txt`, пример окружения — `backend/.env.example`.

Запуск ASGI в проде: `daphne -b 0.0.0.0 -p 8000 server.asgi:application`
(или `uvicorn server.asgi:application`).

## Ограничения

- В режиме разработки — `runserver` + SQLite + in-memory channel layer (один процесс).
  Для масштабирования — PostgreSQL + Redis (см. `ДЕПЛОЙ.md`).
- Загружаемые файлы — до 25 МБ; аватары/стикеры остаются компактными data-URL.
- Звонки требуют интернета (брокер PeerJS + STUN); оба собеседника должны быть онлайн.

## Файлы

- `index.html`, `styles.css`, `app.js` — фронтенд.
- `api.js` — связь фронтенда с сервером (REST + WebSocket).
- `backend/` — Django-проект:
  - `server/` — настройки, `urls.py`, `wsgi.py`, `asgi.py` (WebSocket-роутинг)
  - `api/` — модели, API (`views.py`), WebSocket (`consumers.py`, `routing.py`, `ws_auth.py`),
    рассылка (`notify.py`)
  - `media/` — загруженные файлы, `db.sqlite3` — база
- `backend/test_api.py` — автотест API.
- `backend/Запустить сервер.bat` — запуск бэкенда в один клик.
- `backend/requirements.txt`, `backend/.env.example`, `backend/ДЕПЛОЙ.md` — для продакшена.
